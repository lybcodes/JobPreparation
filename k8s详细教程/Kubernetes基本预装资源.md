# 第三章 安装 ingress-nginx

## 3.1 概述

ingress-nginx 是 k8s 平台的七层负载均衡组件。

> 注意：
>
> - 部署 ingress 的 node 节点会自动开启节点的 `80和443` 端口，保证这个机器端口不会被占用。
>
> - 默认 ingress-nginx在每个节点没有 CPU、MEMORY 最大配额限制；可以按照公司架构需求修改 resoources.limits 相关字段。

## 3.2 安装

给集群中需要暴露的 ingress-nginx 的机器节点上打上 `node-role=ingress` 标签，如：

```shell
kubectl label node k8s-node1 node-role=ingress
kubectl label node k8s-node2 node-role=ingress
kubectl label node k8s-node3 node-role=ingress
```

# 第四章 安装Helm

## 4.1 概述

Helm是 Kubernetes 的包管理工具，就像 Linux 下的包管理器，如：yum、apt 等，可以很方便的将之前打包好的 yaml 文件部署到 Kubernetes 上。

## 4.2 安装

下载：

```shell
wget https://get.helm.sh/helm-v3.6.3-linux-amd64.tar.gz
```

解压：

```shell
tar -zxvf helm-v3.6.3-linux-amd64.tar.gz
```

移动到指定目录：

```shell
mv linux-amd64/helm /usr/local/bin/helm
```

命令补全：

```shell
helm completion bash | sudo tee /etc/bash_completion.d/helm > /dev/null
source /usr/share/bash-completion/bash_completion
```
